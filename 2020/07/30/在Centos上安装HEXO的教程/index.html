在Centos上安装HEXO的详细教程

在Centos上安装HEXO的超详细教程 小白也能用

Hexo介绍

Hexo 是台湾国人开发, 具有Node.js带来的超快生成速度, 几秒内就可以完成百个页面的渲染, 同时支持Markdown文件, 适合写技术Blog

安装前提

  1. Node.js(建议12.0及以上)
  2. Git
  3. 配置好yum的centos7服务器

安装Git(确保配置好了yum源)

使用yum安装git-core 会有点慢

yum -y install git-core

顺带安装可能用到的工具

yum -y install gcc gcc-c++ curl wget vim

为什么要首先要安装 git？ 因为 Hexo 开源且基于 Node.js 环境，所以大量主题和设置需要通过 git 完成

安装Node.js(必装)

安装前的知识补充

Node.js就是JavaScript的运行环境, 类似于java的运行环境jdk

nvm是Node.js的版本控制工具

npm是Node.js的包管理工具(你也可以使用开源的cnpm工具)

附: 淘宝的Node.js下载库地址(本文不需要用到)

https://npm.taobao.org/mirrors/node/

开始安装

这里我使用nvm来安装node.js

第一步,下载nvm控制器

git clone https://github.com/cnpm/nvm.git  # 克隆nvm库到本地
cd nvm   # 进入nvm目录
sh ./install.sh  # 安装nvm
source $HOME/nvm/nvm.sh   # 加载nvm的运行环境, 如果不加载,后面的命令就无法使用,后面会教你怎么样不用每次都加载一遍

第二部, 用nvm下载node.js

nvm ls-remote    # 查看可用版本的命令
nvm install v12.0.0   #安装 12.0.0版本的node.js
nvm use v12.0.0     #选择使用刚才安装的版本

安装Hexo

npm是随同NodeJS一起安装的, 我们可以直接使用

npm install -g hexo --save  # 安装hexo模块

–save、-S参数意思是把模块的版本信息保存到dependencies（生产环境依赖）中，即你的package.json文件的dependencies字段中

-g 全局安装

本地安装

  1. 将安装包放在 ./node_modules 下（运行 npm 命令时所在的目录），如果没有 node_modules 目录，会在当前执行 npm 命令的目录下生成 node_modules 目录。
  2. 可以通过 require() 来引入本地安装的包。

全局安装

  1. 将安装包放在 /usr/local 下或者你 node 的安装目录。
  2. 可以直接在命令行里使用。

部署hexo

mkdir /home/wwwroot    # 创建用来存储hexo生成的网站的文件夹
hexo init /home/wwwroot  # 初始化文件夹

配置hexo

cd /home/wwwroot
ls

可以看到hexo产生的文件

[root@localhost wwwroot]# ls
_config.yml   package.json       scaffolds  themes
node_modules  package-lock.json  source

几个重要文件:

  • _config.yml配置文件
  • themes主题文件夹
  • source存放网站素材的地方
vim _config.yml  #编辑配置文件

至于配置文件中每个配置项目的意思可以参考官方文档https://hexo.io/zh-cn/docs/configuration

网站项

参数 描述
title 网站标题
subtitle 网站副标题
description 网站描述
keywords 网站的关键词。支援多个关键词。
author 您的名字
language 网站使用的语言。对于简体中文用户来说，使用不同的主题可能需要设置成不同的值，请参考你的主题的文档自行设置，常见的有 zh-Hans和 zh-CN。
timezone 网站时区。Hexo 默认使用您电脑的时区。请参考 时区列表 进行设置，如 America/New_York, Japan, 和 UTC 。一般的，对于中国大陆地区可以使用 Asia/Shanghai。

网址项

参数 描述 默认值
url 网址
root 网站根目录
permalink 文章的 永久链接 格式 :year/:month/:day/:title/
permalink_defaults 永久链接中各部分的默认值
pretty_urls 改写 permalink 的值来美化 URL
pretty_urls.trailing_index 是否在永久链接中保留尾部的 index.html，设置为 false 时去除 true
pretty_urls.trailing_html 是否在永久链接中保留尾部的 .html, 设置为 false 时去除 (对尾部的 index.html无效) true

修改hexo主题

这里用fluid主题举例子

fluid主题的git地址:https://github.com/fluid-dev/hexo-theme-fluid

下载主题(网络原因,这个时间可能会有点长⌛)

cd /home/wwwroot/themes            # 进入主题目录
git clone https://github.com/fluid-dev/hexo-theme-fluid fluid                                                               #  克隆主题库并且改名为fluid
ls                              # 查看下载的主题

引用主题

vim /home/wwwroot/_config.xml    #修改默认主题为fluid
 85 updated_option: 'mtime'
 86 
 87 # Pagination
 88 ## Set per_page to 0 to disable pagination
 89 per_page: 10
 90 pagination_dir: page
 91 
 92 # Include / Exclude file(s)
 93 ## include:/exclude: options only apply to the 'source/' folder
 94 include:
 95 exclude:
 96 ignore:
 97 
 98 # Extensions
 99 ## Plugins: https://hexo.io/plugins/
100 ## Themes: https://hexo.io/themes/
101 theme: fluid    #修改为我们克隆的主题的文件夹名字
102 
103 # Deployment
104 ## Docs: https://hexo.io/docs/deployment.html
105 deploy:
106   type: ''

按一下esc,在输入:后,输入wq,回车,即可保存并退出

启动Hexo

hexo new "This is a test page" # 新建一个网页页面,会在/home/wwwroot/source/_posts/中生成对应md文件, 你可以自己在其中书写内容
hexo g      #使用hexo生成网页
hexo c      #清除hexo的配置(常用在修改主题,网站配置后, 修改网页内容一般不用)

我通常的通过上传md文件, 再用hexo g命令重新生成页面来实现文章的上传

我们来看一下hexo new “This is a test page”生成的md文件内容

[root@localhost themes]# cat /home/wwwroot/source/_posts/test.md
---
title: test
date: 2020-07-30 18:52:31
tags:
---

所以, 我们只要在写md的时候加入上面的头部格式就可以了

使用nginx发布Hexo生成的网页

前面我们已经使用Hexo生成了静态网页

现在我们需要将它发布到网上, 这里我使用的是nginx服务器, 你也可以使用apache服务器

yum -y install nginx

然后我们修改nginx服务器的网站根目录为/home/wwwroot/public

按一下esc,在输入:后,输入wq,回车,即可保存并退出

下面是重点内容:

要想nginx被外网访问需要关闭firewalld防火墙, 关闭selinux(有些服务器厂商一般默认关闭), 如果你购买的是阿里云这些服务器, 需要设置阿里云自己的防火墙, 开放其中的80端口, 否则无法被访问到

关闭firewalld并清空防火墙规则

systemctl stop firewalld
systemctl disable firewalld
iptables -F
iptables -X
iptables -Z

关闭selinux

setenforce 0

永久关闭要修改文件

vim /etc/sysconfig/selinux
 1 
 2 # This file controls the state of SELinux on the system.
 3 # SELINUX= can take one of these three values:
 4 #     enforcing - SELinux security policy is enforced.
 5 #     permissive - SELinux prints warnings instead of enforcing.
 6 #     disabled - No SELinux policy is loaded.
 7 SELINUX=disabled   #修改为disabled,重启有效
 8 # SELINUXTYPE= can take one of three values:
 9 #     targeted - Targeted processes are protected,
10 #     minimum - Modification of targeted policy. Only selected processes are protected. 
11 #     mls - Multi Level Security protection.
12 SELINUXTYPE=targeted

好了,现在启动nginx

systemctl start nginx

通过ip地址或者域名就可以访问到你的Blog网站啦
