LNMP搭建与个人博客的部署

LNMP搭建与个人博客的部署

primary

环境规划

IP 主机名 节点
192.168.100.100 master LNMP平台

配置基础环境

关闭防火墙和SElinux

systemctl stop firewalld
systemctl disable firewalld
setenforce 0

配置IP地址

cd /etc/sysconfig/network-scripts/ ## 进入网卡配置文件夹
vi ifcfg-eno16777736  ## 编辑网卡文件
TYPE=Ethernet
BOOTPROTO=static  ## 修改为静态IP地址
DEFROUTE=yes
PEERDNS=yes
PEERROUTES=yes
IPV4_FAILURE_FATAL=no
IPV6INIT=yes
IPV6_AUTOCONF=yes
IPV6_DEFROUTE=yes
IPV6_PEERDNS=yes
IPV6_PEERROUTES=yes
IPV6_FAILURE_FATAL=no
NAME=eno16777736
UUID=751ec73f-6cff-486d-93cf-1ddf3d088180
DEVICE=eno16777736
ONBOOT=yes
IPADDR=192.168.100.100    ## 配置IP
NETWORK=255.255.255.0     ## 配置网络掩码

wq保存退出

systemctl restart network ## 重启网络服务

在宿主机验证网络配置是否正确

ping 192.168.100.100

无法ping通:

  • 检查宿主机仅主机模式网卡是否开启
  • 检查vmware是否连接仅主机模式网卡

使用CRT连接虚拟机, 进行后续操作

配置YUM源

备份官方yum配置文件

mkdir /root/backup    ## 创建一个用于存放备份文件的文件夹
cd /etc/yum.repos.d/  ## 进入yum配置文件存放文件夹
mv ./* /root/backup   ## 备份

配置本地yum源(等同离线版应用商店)

vi local.repo ## 必须以.repo结尾
[local]
name=local
baseurl=file:///media/cdrom
gpgcheck=0
enabled=1

[lnmp]
name=lnmp
baseurl=file:///root/lnmp
gpgcheck=0
enabled=1

挂载镜像文件(/media/cdrom)

可以看到镜像文件CentOS-7-x86_64-DVD-1511.iso和lnmp都在root的家目录下

[root@localhost ~]# ls
anaconda-ks.cfg               Docker.tar.gz  wordpress-4.7.3-zh_CN.zip
CentOS-7-x86_64-DVD-1511.iso  K8S.tar.gz     xiandian
cirros-0.3.4-x86_64-disk.img  lnmp

挂载

[root@localhost ~]# mkdir /media/cdrom/  ## 创建用于挂载的文件夹
[root@localhost ~]# mount /root/CentOS-7-x86_64-DVD-1511.iso /media/cdrom/
mount: /dev/loop1 is write-protected, mounting read-only

(其他lnmp本身就是本地的文件夹, 所以无挂载可言)

检查yum源配置

yum list

若只打印出许多rpm软件包的名字则配置正确

[root@localhost ~]# yum repolist
Loaded plugins: fastestmirror
Repodata is over 2 weeks old. Install yum-cron? Or run: yum makecache fast
Loading mirror speeds from cached hostfile
repo id                               repo name                           status
!lnmp                                 lnmp                                  178
!local                                local                               3,723

至此基础配置完成, 再做一次快照保存当前的状态

接下来开始LNMP平台的搭建

安装mariadb

mariadb是mysql的分支

yum -y install mariadb mariadb-server

安装过程可能有点慢

  • mariadb 数据库的客户端程序(通过这个程序可以输入sql语句)
  • mariadb-server 数据库服务程序

启动mariadb服务程序

systemctl start mariadb

初始化mariadb

mysql_secure_installation

交换界面操作:

直接回车

输入y +回车

输入root(mariadb的root用户)密码000000

再次输入密码

输入y +回车

输入n+回车

输入y +回车

输入y +回车

登录mariadb

mysql -uroot -p000000  ## 注意空格的位置
[root@localhost ~]# mysql -uroot -p000000  
Welcome to the MariaDB monitor.  Commands end with ; or \g.
Your MariaDB connection id is 2

Type 'help;' or '\h' for help. Type '\c' to clear the current input statement.

MariaDB [(none)]>

使用Ctrl+C或者输入quit退出交换界面

安装php

需要安装的软件包

  • php
  • php-fpm(php与nginx通信用)
  • php-mysql(php与maraidb通信用)
yum -y install php php-fpm php-mysql

编辑php配置文件

vi /etc/php-fpm.d/www.conf

末行模式输入set nu可以显示行号

38 ; RPM: apache Choosed to be able to access some dir as httpd
39 user = nginx  # 修改用户为nginx
40 ; RPM: Keep a group allowed to write in log dir.
41 group = nginx  # 修改用户组为nginx

输入wq保存退出

启动php-fpm

systemctl start php-fpm

安装nginx

nginx等同于Apache | IIS

安装

yum -y install nginx

配置nginx

指定网站根目录

mkdir /www ## 创建网站根目录文件夹
chown nginx:nginx -R /www  ## 更改根目录文件夹的宿主为nginx用户

编辑配置文件

vi /etc/nginx/conf.d/default.conf
 8     location / {
 9         root   /www;    ## 修改根目录
10         index  index.html index.htm index.php;  ## 添加index.php文件为默认页面
11     }
30     location ~ \.php$ {         ## 取消30到36行前面的注释"#"
31         root           html;
32         fastcgi_pass   127.0.0.1:9000;
33         fastcgi_index  index.php;
34         fastcgi_param  SCRIPT_FILENAME  /www$fastcgi_script_name;   ## 修改此行
35         include        fastcgi_params;
36     }

启动nginx

systemctl start nginx

测试环境

进入网站根目录

cd /www

创建测试文件

echo "<?php phpinfo();" > index.php

使用浏览器输入IP地址访问

出现上面的页面就说明php与nginx均正确安装

如果访问nginx提示被拒绝, 可以查看Linux的防火墙是否关闭

Linux中, 一切皆文件
大部分配置文件的修改都不会被直接应用, 需指定系统重新加载配置文件才被应用(如:重启服务)
