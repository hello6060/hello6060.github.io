配置主从DNS服务器

配置主从DNS服务器

安装主DNS

参考 搭建DNS服务器

其中区域配置文件的/etc/named.rfc1912.zones改为

zone "ljh.com" IN {
        type master;
        file "named.localhost";
        allow-update { 10.10.10.11; };    # 添加从服务器的地址, 两边有空格
};

zone "10.10.10.in-addr.arpa" IN {
        type master;
        file "named.loopback";
        allow-update { 10.10.10.11; };    # 添加从服务器的地址, 两边有空格
};

重启主DNS

systemctl restart named

安装从DNS

修改主配置文件

修改区域配置文件

zone "localhost.localdomain" IN {
        type slave;
        file "slaves/named.localhost";    //数据文件保存于slaves文件夹中
        masters { 10.10.10.12; };
        // 要删除allow-update { none; };
};

zone "1.0.0.127.in-addr.arpa" IN {
        type slave;
        file "slaves/named.loopback";    //数据文件保存于slaves文件夹中
        masters { 10.10.10.12; };
        // 要删除allow-update { none; };
};

重启DNS

systemctl restart named

可以在从服务器查看到同步的数据文件

[root@DNS2 ~]# ls /var/named/slaves/
named.localhost  named.loopback

测试

配置测试机

[root@DNS2 ~]# vim /etc/resolv.conf
# Generated by NetworkManager
search localdomain ljh.com
nameserver 10.10.10.11             # 指向从DNS服务器

重启测试机网络服务

[root@DNS2 ~]# systemctl restart network

你还可以用个dig命令直接指定DNS服务器, 这样就不用修改/etc/resolv.conf了
