Docker容器编排[Docker compose]

Docker容器编排[Docker compose]

介绍

Docker-Compose是一个容器编排的工具，适用于所有环境：生产、开发、测试以及CI工作流程

安装Docker-compose

工作流程要求:

①构建一个基于Python3.4的镜像。
②把当前目录添加到镜像中的/code路径下。
③把工作路径设置成/code。
④安装Python依赖。
⑤设置容器的默认命令为pythonapp.py。

wget https://github.com/docker/compose/releases/download/1.26.2/docker-compose-Linux-x86_64

添加到系统任意一个bin文件夹(PATH常量中的任意)

mv docker-compose-Linux-x86_64 /usr/local/bin/docker-compose

给权限

chmod +x /usr/local/bin/docker-compose

使用compose

创建实验文件目录

编写实验文件

mkdir composetest
cd !$

编写实验用python文件app.py

import time

import redis
from flask import Flask

app = Flask(__name__)
cache = redis.Redis(host='redis',port=6379)

def get_hit_count(): 
    retries=5
    while True:
        try:
            return cache.incr('hits')
        except redis.exceptions.ConnectionError as  exc:
            if retries == 0:
                raise exc
            retries -= 1
            time.sleep(0.5)

@app.route('/')
def hello():
    count = get_hit_count()
    return 'HelloWorld!I have been seen {} times.\n'.format(count)
if __name__ == "__main__":
    app.run(host="0.0.0.0",debug=True)

在这个例子中，Redis就是应用网络中Redis容器的主机名。端口使用的Redis默认端口6379。

创建requirements.txt文件

[root@master composetest]# vim requirements.txt

flask
redis

创建Dockfile文件

在这一步中，需要编写一个Dockerfile来构建一个Docker镜像。这个镜像包含Python应用的所有依赖，也包含Python其本身。

[root@master composetest]# vim Dockerfile

FROM python:3.4-alpine
ADD . /code
WORKDIR /code
RUN pip install -r requirements.txt --default-timeout=100
CMD ["python","app.py"]

–default-timeout=100加大下载延迟, 防止报错

创建docker-compose.yml文件, 定义服务

[root@master composetest]# vim docker-compose.yml

version: '3'
services:
        web:
                build: .
                ports:
                 - "5000:5000"
        redis:
                image: "redis:alpine"

这个Compose文件中定义了两个服务Web与Redis。

Web服务使用当前目录Dockerfile构建出来的镜像，并且将容器上暴露的5000端口转发到主机的5000端口，使用FlaskWeb服务器的默认端口5000。

Redis服务使用从DockerHub注册表中拉取的公有镜像。

停止服务

可以在项目路径下使用docker-composedown命令停止服务，也可以在终端上按Ctrl+C键停止当前启动着的应用。

将数据持久化

编辑docker-compose.yml文件, 让容器目录映射到宿主机目录

[root@master test]# vim docker-compose.yml 

version: '3'
services:
        web:
                build: .
                ports:
                 - "5000:5000"
                volumes:
                 - .:/code
        redis:
                image: "redis:alpine"

新的volumes键将当前路径（项目路径）与容器中的/code路径挂载到一起，允许用户及时修改代码而不用重新构建镜像。

重建服务

docker-compose up

可以看到上次开启服务的记录保留着

因为应用程序的代码通过volume挂载到容器中，你可以更改其代码并立即查看更改，而不必重新生成镜像。

vim app.py

return 'Hello Worlddddddddddddddd! I have been seen {} times.\n'.format(count)
